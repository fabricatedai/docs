The Problem

Time-to-value for Physical AI is unacceptable

Partner conversations reveal 3 key problems impacting time to value: integration is bespoke for every device, agents have no way to discover or control physical systems, and security for agent-to-device interaction is unsolved. Three problems compound.

Problem 1

Integration is bespoke

Every device ships with its own SDK and protocol. A robotic arm speaks ROS 2, a camera uses RTSP, a sensor uses MQTT. Coordinating them means custom glue code for each pairing per site. Brittle, hard to maintain, breaks when anything changes. Each new location is a re-integration project and cost scales linearly with deployment count.

Problem 2

Agents lack device awareness

Agentic frameworks were built for cloud reasoning, not physical device coordination. IoT platforms were built for telemetry and provisioning. Neither lets an AI agent dynamically discover a device, call a function, and enforce safety before execution. No standard layer bridges the gap.

Problem 3

Agents introduce new security risks

Traditional IoT security protects the wire. AI agents introduce new attack vectors: hallucination-to-actuation, agent credential escalation, multi-tenant authority conflicts, and trust in agent-generated code. A robot arm that receives a bad command moves. It might not ask for confirmation.

The Solution

The missing layer between agents and physical devices

Device Connect is an open-source Python framework from Arm that lets any AI agent discover and invoke functions on any physical device. Framework-agnostic, works device-to-device with zero infrastructure, and scales to thousands.

Capability 1

Discover

Devices self-register with a simple Python driver, announcing capabilities, state, and available functions. Agents adapt to the fleet in real time with no hardcoded knowledge of what is deployed at a site.

Capability 2

Invoke

Any AI agent can call any function on any device using four core tools: discover_devices(), invoke_device(), get_device_status(), and subscribe(). Works across Strands, LangGraph, Claude MCP, or plain Python.

Capability 3

Govern

Every action passes through policy checks before execution on the device itself, sub-millisecond, no cloud round-trip. Capabilities-based authorization where permissions travel with the requestor. Full audit logging.
